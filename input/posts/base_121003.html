I've had this idea for a while, Unicode as a numeral system , or more specifically using as much of the characters in the standard as possible to represent numbers in a base of the size of that set of characters. This is by no means usable in real life, not only because of the size (121,003 was the final number of "digits" I came up with), but because Unicode has a lot of characters that either look similar or are the same.

When making the converter, the real problem was determining the set of the characters to use, because Unicode is complicated. To convert to my base Unicode, I needed a list that a program could use. My first stop was the Unicode Character Database where I got a XML version of the database. After wrangling with python's XML module for the first time, I finally got it to get the items after I realized I needed to include the xml namespace for the ucd.

My intial critria was to ignore anything that wasn't printable (not control characters like newline ('\n' 000A) or null ('\0' 0000)). This was expanded to any character that doesn't have a name.
